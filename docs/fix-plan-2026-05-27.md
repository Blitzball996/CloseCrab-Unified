## 问题清单（按优先级）

### P0: #7 子 agent 超时闪退

**根因**: `AgentManager.cpp:125` 的 `instance->future.wait()` 无超时无限阻塞  
**影响**: Agent 卡死时主线程永远等待，用户强制退出导致进程崩溃  
**相关文件**:
- `src/agents/AgentManager.cpp` (第 116-130 行)
- `src/agents/AgentManager.h`

**JackProAi 方案**: 独立 AbortController + finally 清理块 + killShellTasksForAgent  
**修复方案**:
1. `future.wait()` → `future.wait_for(std::chrono::seconds(300))`
2. 超时后标记 `status = FAILED`, `error = "Agent timed out after 300s"`
3. 添加 `catch(...)` 兜底防止未知异常崩溃进程
4. 添加 `cancelAgent(agentId)` 方法允许外部取消
5. 析构函数中清理所有未完成 agent（wait_for 5s 后放弃）

---

### P1: #8 对话没有被保存

**根因**: `SessionAutoSave::save()` 定义了但**从未被调用**，context 永远是初始值 `{}`  
**影响**: `/resume` 看到槽位但内容为空  
**相关文件**:
- `src/services/SessionAutoSave.h` (save 方法已实现)
- `src/main.cpp` (需要添加调用)
- `src/commands/SessionCommands.h` (恢复逻辑，检查 `context == "{}"`)

**JackProAi 方案**: 每个事件自动 POST 到 Bridge API + 本地文件缓存  
**修复方案**:
1. 在 `callbacks.onComplete` lambda 中调用 `SessionAutoSave::save(&sessionMgr, sessionId, g_queryEngine->getMessages())`
2. 用 try-catch 包裹，失败只 log 不 crash
3. 添加 `#include "services/SessionAutoSave.h"` 到 main.cpp
4. lambda capture 中加入 `&sessionMgr, &sessionId`

---

### P2: #3 Retrying UX

**根因**: `APIError.h` 的 `withRetry()` 中重试只通过 `spdlog::warn()` 输出到日志文件，UI 无显示  
**影响**: 用户看不到重试进度，以为程序卡死  
**相关文件**:
- `src/api/APIError.h` (第 72-100 行, withRetry 模板函数)

**JackProAi 方案**: React 组件实时显示 `"Retrying in Xs... (attempt N/M)"`  
**修复方案**:
1. 添加 `using RetryCallback = std::function<void(int attempt, int maxRetries, const std::string& error)>`
2. 给 `withRetry()` 添加可选第三参数 `RetryCallback onRetry = nullptr`
3. 重试时调用 `onRetry(attempt, maxRetries, e.what())`
4. 调用方传入回调更新 spinner: `spinner.setMessage("retrying (3/8, waiting 8s...)")`

### P2: #4 绿色 OK 不明所以

**根因**: `main.cpp:693` 对所有非 Bash 工具统一打印绿色 `"OK"`  
**影响**: 用户不知道工具做了什么  
**相关文件**:
- `src/main.cpp` (callbacks.onToolResult lambda)

**JackProAi 方案**: 每个工具有 `renderToolResultMessage()` 显示完整内容  
**修复方案**:
按工具类型显示具体结果：
- Read → `✓ filename (N lines)`
- Write → `✓ wrote filename (N bytes)`
- Edit → `✓ edited filename (description)`
- Glob → `✓ N files matched`
- Grep → `✓ N matches in M files`
- WebSearch → `✓ N results`
- WebFetch → `✓ fetched url`
- Agent → `✓ agent spawned: id`
- 其他 → `✓ toolName: first 80 chars of result`
- Bash/PowerShell → 保持现有折叠输出逻辑不变

### P2: #5 看不到写了什么代码

**根因**: FileWriteTool 返回 `"File written: path (size bytes)"`，FileEditTool 返回 `"Replaced lines X-Y in path"`，不含内容  
**影响**: 用户看不到实际修改了什么  
**相关文件**:
- `src/tools/FileWriteTool/FileWriteTool.h` (第 54 行)
- `src/tools/FileEditTool/FileEditTool.h` (第 108-109, 173 行)

**FileEditTool 修复**: 执行替换后附加 diff 预览（`- old前3行 / + new前3行`）
**FileWriteTool 修复**: 写入后附加前 20 行内容预览（每行 max 120 字符）

## API 503 问题

**现象**: 频繁报 `error api temporarily unavailable`
**根因**: 代理限流 + MAX_RETRIES=3 太少 + 延迟太短 + 重试静默
**修复** (`src/api/RemoteAPIClient.cpp`):
- MAX_RETRIES 3 → 8
- 延迟 2s/4s/8s/16s/30s
- 日志 debug → warn
- fallback_model 设为 sonnet
