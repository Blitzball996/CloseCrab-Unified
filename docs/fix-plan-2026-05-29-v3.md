## 1. 缓存断点策略 (prompt cache)

### JackProAi 实际做法

**文件**：`services/api/claude.ts`

- **systemBlocks**（`buildSystemPromptBlocks` line 3213）：调用 `splitSysPromptPrefix`（`utils/api.ts:321`）把 system 拆成
  - 归因头（不缓存）
  - 静态前缀（CLI 内置 prompt，`cacheScope='global'`）
  - 动态后缀（`cacheScope=null`）
  
  用 `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 标记把"会变"和"不会变"明确切开。
  
- **tools**（line 1396 `allTools = [...toolSchemas, ...extraToolSchemas]`）：
  - cache_control 标记放在 `toolSchemas` 的最后一个工具（line 1388 注释："toolSchemas (which carries the cache_control marker) so toggling /advisor only churns the small suffix"）
  - 任何 advisor/动态扩展工具 **追加在 cache_control 之后**，扰动不影响缓存
  - `defer_loading` 工具从 cache hash 中**完全排除**（line 1465-1467："Exclude defer_loading tools from the hash"）
  
- **messages**（`addCacheBreakpoints` line 3063）：
  - **整个请求只放 1 个 message-level cache_control**（line 3078 注释："Exactly one message-level cache_control marker per request")
  - 标记位置：`markerIndex = messages.length - 1`（last message）；fork 模式 `skipCacheWrite=true` 时挪到 `messages.length - 2`

### CloseCrab 当前

**文件**：`src/api/RemoteAPIClient.cpp` line 35-100

```cpp
// 1. system 全文一个 cache_control（line 38）
body["system"] = nlohmann::json::array({
    {{"type", "text"}, {"text", systemPrompt}, {"cache_control", ...}}
});

// 2. 最后一条消息最后一个 block 一个 cache_control（line 83-88）
lastMsg["content"].back()["cache_control"] = {{"type", "ephemeral"}};

// 3. tools 最后一个 cache_control（line 93-98）
tools.back()["cache_control"] = {{"type", "ephemeral"}};
```

**问题**：
- system 没拆静态/动态。MEMORY.md、cwd、当前模型名混在一起。`buildSystemPrompt()` 用 `cachedSystemPrompt_` 缓存（QueryEngine.cpp:60-65）保证字节稳定，但只要 `currentModel` 变一次 → 整个 system 缓存失效。
- tools 放在 `tools.back()`。CloseCrab 的 defer_loading 实现（QueryEngine.cpp:210-280）让 tools 数组每轮可能扩容 → 最后一个 tool 变了 → 整个 tools 前缀缓存失效。
- 3 个断点 ≤ Anthropic 上限 4 个，**数量没爆**；问题是放在了易变位置。

### 改法

| 文件:行 | 改 |
|---|---|
| `src/core/QueryEngine.cpp:60-176` | 把 `buildSystemPrompt()` 拆成 `buildStaticPrefix()` + `buildDynamicSuffix()`。静态：内置 prompt + CLAUDE.md + MEMORY.md。动态：cwd / 当前模型 / 时间戳 / plan/coord 模式。`config_.systemPrompt` 内 cwd 那行（line 126-127）挪到动态。|
| `src/api/RemoteAPIClient.cpp:36-40` | system 改成数组传 2 个 block：`[{text: staticPrefix, cache_control: ephemeral}, {text: dynamicSuffix}]`。动态那块不带 cache_control。|
| `src/core/QueryEngine.cpp:210-280` | **不再"按发现动态扩展 tools"**。固定 tools = `ALWAYS_LOAD` 集合（17 个核心）+ 所有 always-on，按 JackProAi 那样 `defer_loading=true` 标记其余 ~40 个 MCP/扩展工具。defer_loading 字段就算中转代理不识别也只会被剥掉（与缓存兼容）。|
| `src/api/RemoteAPIClient.cpp:93-100` | 只在**未标 defer_loading** 的 tools 末尾打 cache_control。把 defer_loading 的工具排在数组后面，使前缀稳定。|
| `src/api/RemoteAPIClient.cpp:83-88` | 把 messages 最后一条的 cache_control 改成"last message" 1 个，且**只在 last message 是 user 角色时**打（assistant block 通常以 thinking/tool_use 结尾，按 JackProAi `assistantMessageToMessageParam` line 658-660 跳 thinking）。|

**预期**：第二轮起 `cache_read_input_tokens ≥ system_static + tools_stable ≈ 60K`，写量降到 5–10K 增量。账单从 $7.38 → < $1。


## 5. rm/危险命令拦截

### JackProAi

**两层防御**：

1. **路径黑名单**（`utils/permissions/pathValidation.ts:331` `isDangerousRemovalPath`）
   ```ts
   // 拒绝的路径：
   - "*" 或 endsWith("/*")  // rm /path/dir/*
   - "/" 根目录
   - WINDOWS_DRIVE_ROOT_REGEX: /^[A-Za-z]:\/?$/  // C:\
   - WINDOWS_DRIVE_CHILD_REGEX: /^[A-Za-z]:\/[^/]+$/  // C:\Users
   - homedir()
   - 任何根目录的直接子目录: /usr, /tmp, /etc
   ```

   ```ts
   - git checkout . / git restore . → "may discard all working tree changes"
   - DROP/TRUNCATE TABLE → "may drop or truncate database objects"
   - kubectl delete / terraform destroy → ...
   ```
   命中 → 在确认对话框里显示警告字符串。
