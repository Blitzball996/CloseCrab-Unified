## C. PDF（最大的"真"差距）

| # | 功能 | claude-code 参照 | CloseCrab 当前 | 状态 | 说明 |
|---|------|------|------|:---:|------|
| C1 | 页码范围 `1-5/3/10-` | — | 解析支持 | ✅ | |
| C2 | 最多 20 页保护 | — | 有 | ✅ | |
| C3 | 文本提取 | — | poppler 或外部 pdftotext | ✅ | 纯文字 PDF 够用 |
| C4 | **原生 PDF 视觉（图/表/扫描件）** | **送 document block 给 API，模型直接"看" PDF** | **只做文本提取** | ❌ | **poppler 也补不上**——我们丢掉了图、表、扫描页 |
| C5 | 加密 PDF 检测 | — | poppler 路有 / 外部路无 | 🟡 | |
| C6 | 零依赖（不装 poppler/pdftotext） | — | 关 poppler 时需用户装 pdftotext | 🟡 | 这就是"无需下载"的破口 |

> **关键诚实点**：你问"开 poppler 是不是 100%对齐"——**不是**。poppler 只把 **C3 文本提取质量**从"调外部进程"提到"原生库"。但 claude-code 的 PDF 是**把整个 PDF 发给 API 让模型原生看**（含图表/扫描），我们走的是**文本提取**路线，**架构不同**，poppler 关不上 C4。要真对齐 C4，得改成"PDF→base64 document block 发给 API"，且要 API/中转支持。

## 汇总：还有哪些"真"没对齐（按优先级）

| 优先级 | 缺口 | 影响 | 能否补 |
|:---:|------|------|------|
| 🔴 高 | **E4 权限系统没接线** | 安全特性完全不生效 | 能补：call() 里加 isReadDenied + main 里 loadPermissions |
| 🟠 中 | **C4 PDF 原生视觉** | 扫描件/图表 PDF 读不到内容 | 能补但要改架构（发 document block）+ 看中转支持 |
| 🟠 中 | **B5 WebP 不解码** | WebP 大图发不出 | 能补：换全功能 stb 编译选项，或加 libwebp |
| 🟡 低 | A4 单行超长不截 | 极长单行可能膨胀 context | 易补 |
| 🟡 低 | G2 图片/PDF 不写 readFileState | 罕见写场景 | 易补 |
| ⚪ 不补 | B6/B7/B8 智能裁剪/MozJPEG/滤镜 | 无 | 伪需求，参照物也没有 |
